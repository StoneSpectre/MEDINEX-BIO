# Volume II: Biomedical Infrastructure

## Chapter 5: Metadata & Data Governance

As the Bioquora data lake swells to petabyte scale—housing everything from public ontologies to highly sensitive human genomic data—simply storing the data is insufficient. Without a rigorous, mathematically enforced governance architecture, the data lake inevitably degrades into a "Data Swamp." 

If a data scientist cannot trust the origin of a dataset, or cannot decipher what the column `score_v2` actually means, the data is functionally useless. Bioquora implements a strict Metadata and Governance architecture to manage this complexity.

### 5.1 The Master Data Catalog
A Data Catalog acts as the searchable, centralized inventory of all data assets within the Bioquora ecosystem. It allows engineers, data scientists, and ML algorithms to discover what datasets exist without having to manually query the raw Parquet files in S3.

*   **Implementation:** Bioquora utilizes enterprise-grade cataloging tools (such as **DataHub** or **Amundsen**). 
*   **Asset Registration:** When a new Airflow DAG generates a Parquet table, the DAG automatically makes an API call to the Data Catalog, registering the new asset.
*   **Rich Metadata:** The catalog stores technical metadata (column names, data types, row counts), operational metadata (when the table was last updated, who owns the pipeline), and business metadata (descriptions of what the table is used for).

### 5.2 Enforcing Data Contracts
A major source of system failure occurs when a data producer unexpectedly changes a schema. If the team responsible for ingesting ClinVar data changes the column `variant_id` to `clinvar_id`, the downstream ML pipelines training on that data will crash.

**The Solution: Data Contracts**
Data Contracts are formal, machine-enforceable agreements between the data producers and the data consumers.
*   The contract (written in YAML) specifies the exact schema, the allowed data types, the expected data quality bounds (e.g., `< 1% nulls`), and the Service Level Agreement (SLA) for uptime.
*   Before any code change to an ETL pipeline can be merged into the `main` branch, the CI/CD pipeline tests the code against the Data Contract. If the change violates the contract, the deployment is blocked. This guarantees downstream ML models are never poisoned by unexpected schema mutations.

### 5.3 Automated Provenance and Lineage Tracking
In clinical AI, auditability is non-negotiable. If a patient questions why the Bioquora Phase 1 Risk Engine flagged them as "High Risk" for cardiovascular disease, the engineering team must be able to trace that specific prediction back through the entire mathematical pipeline to the exact raw datasets that trained the model.

*   **Data Lineage** tracks the journey of data from origin to endpoint as a Directed Acyclic Graph (DAG).
*   **Table-Level Lineage:** "The `Patient_Risk_Scores` table was generated by joining the `Demographics` table with the `Biomarkers` table."
*   **Column-Level Lineage:** "The `final_risk_score` column in this table was derived by running the XGBoost Model v1.2 over the `hba1c` column from the source table."
*   **Implementation:** Bioquora uses tools like **OpenLineage** integrated directly into Apache Spark and `dbt`. Every time a transformation executes, OpenLineage emits an event to the Data Catalog mapping exactly what happened.

### 5.4 The Data Dictionary (Business Ontology)
While the Data Catalog tracks physical datasets (tables and columns), the Data Dictionary tracks the philosophical definitions of the data. It maps biological and business logic to technical implementation.

For example, if two different ML teams are building models to predict "Type 2 Diabetes," they must agree on the mathematical definition of that disease. 
The Data Dictionary defines exactly how "Type 2 Diabetes" is codified in the SQL layer (e.g., `WHERE Fasting_Glucose > 126 mg/dL OR HbA1c >= 6.5% OR ICD10_Code = 'E11'`). 
By centralizing these definitions, Bioquora ensures that all AI agents and predictive models are operating on the exact same biological axioms.
